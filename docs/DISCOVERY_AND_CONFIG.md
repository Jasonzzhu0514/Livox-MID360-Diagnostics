# 发现与配置说明

`livox-mid360-diagnostics autoconfig` 是 MID360 配置辅助工具。

主要行为：

- 默认搜索当前目录、用户目录和仓库目录下的 MID360 配置文件；也可以用 `LIVOX_MID360_SEARCH_ROOTS` 指定搜索根目录。
- 如果显式传入 `--config`，只处理指定的配置文件。
- 根据配置中的雷达 IP 和本机 IPv4 网卡选择候选以太网接口。
- 主动扫描本机子网，执行 `ping`，读取 `ip neigh`，并根据 Livox MAC 前缀、高 TTL，以及常见的 `192.168.1.*` MID360 网络布局给候选设备打分。
- 如果主动扫描没有结果，则回退到被动方式：用 `tcpdump` 监听 UDP `56000` 端口和 ARP 包。
- 可选地从最近 ROS 日志、额外抓包结果，或已有缓存的 SDK helper 中补充 broadcast code/SN。
- 打印检测到的接口、IP、SN 和配置状态。
- 交互式终端会显示带颜色的方向键菜单；配置按“推荐更新 / 已匹配 / 其它候选”分组，低优先级示例配置默认折叠。上下方向键移动，空格选择/取消，`a` 显示或隐藏低优先级候选，回车确认后写入雷达 IP 和主机 IP，`q` 或 `Esc` 退出不修改。可用 `--show-all` 默认展示所有候选，用 `--no-color` 或 `NO_COLOR=1` 禁用颜色。
- 非交互式终端只显示检测结果和配置列表，不写文件。脚本化写入需要显式使用 `--config PATH --apply --yes`。
